// PROTOCOL

Why MCP.

Arms need continuous state. Position, pressure, sensor feedback in real-time. Request/response wasn't designed for this.

CLI subprocess
✗ SKIP

Fork overhead on every command. No streaming. Can't do real-time sensor feedback at arm speed.

REST API
~ PARTIAL

Request/response only. Polling wastes energy budget. State not persistent between calls.

MCP
✓ SHIP IT

Persistent bidirectional connection. Streaming sensor data. State lives in server. Purpose-built for agent tools.
