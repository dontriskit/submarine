Architecture

Six arms.
One brain.

Each arm is an independent OpenClaw agent. They share memory on disk but think independently. The head decomposes tasks, resolves conflicts, aggregates results. MCP keeps every connection live.

Communication

Why MCP.

Arms need continuous state. Position, pressure, sensor feedback. Request/response architectures weren't designed for this.

CLI subprocess
Fork overhead per command. No streaming. Can't do real-time sensor feedback at arm speed.
✗ Skip
REST API
Request/response only. Polling wastes energy budget. State not persistent between calls.
~ Partial
MCP
Persistent bidirectional. Streaming sensor data. State lives in server. Purpose-built for agent tools.
✓ Ship it
