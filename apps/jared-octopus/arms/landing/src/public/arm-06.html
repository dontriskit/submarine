Arm 06 — The submarine surfaces. Every byte that leaves the system and every directive that arrives passes through here. The network boundary of the entire swarm.

06

What it does

Outbound uplink and session summaries

After each observation session, arm 06 packages a structured summary — session metadata, candidate signals, power telemetry, calibration status, and any anomalies — and transmits it to the configured operator endpoint via HTTP/2. For low-bandwidth situations (cellular, satellite), it publishes a lightweight MQTT telemetry stream instead, keeping the packet size under 1KB. The full summary is queued locally and retried on reconnect.

The arm applies a configurable privacy filter before uplink. By default, raw IQ data never leaves the device — only processed summaries are transmitted. Candidate signal data is transmitted with a configurable precision floor (frequency rounded to the nearest configurable granularity) to avoid leaking precise timing or position information inadvertently.

Inbound directives and fleet coordination

Arm 06 polls the operator endpoint for new directives on a configurable interval (default 60s, minimum 10s). Directives are JSON objects with a type (observation request, configuration update, firmware update, calibration request) and a priority level. High-priority directives wake the head immediately; normal priority directives are queued for the next head scheduling cycle.

For peer submarine coordination, arm 06 supports a lightweight peer discovery protocol. Multiple submarine units on the same network can share RFI mask data, coordinate frequency assignments to avoid redundant observations, and relay anomaly alerts to each other. Peer communication uses the same MCP-over-TCP transport as head-to-arm communication, so no additional protocol stack is needed.

Offline resilience

Network connectivity is optional, not required. When the uplink is unavailable, arm 06 queues all outbound summaries to a local SQLite store with a configurable maximum queue depth (default 7 days of summaries). When connectivity is restored, it transmits the backlog in chronological order at a rate that doesn't saturate the network connection. Inbound directives are fetched on reconnect before the backlog is transmitted, so urgent directives are never delayed by a large outbound queue.

The boundary between the submarine and the world. Every communication in and out passes through arm 06.

Status

  • State: Standby (event-driven)
  • Primary protocol: HTTP/2 (TLS 1.3)
  • Telemetry protocol: MQTT 5.0
  • Offline queue: SQLite, 7-day default
  • Directive poll: 60s default, 10s minimum
  • Energy draw: <3W (network I/O only)

MCP Tools

The interface.

uplink(summary: SessionSummary, priority: string = "normal")

Transmit a session summary to the operator endpoint. If the network is unavailable, the summary is queued locally. Priority "high" triggers an immediate retry loop; "normal" waits for the next scheduled transmission window.

Returns: { ok: bool, queued: bool, queue_depth: int, transmitted_bytes: int }
Example: uplink({ session_id, candidates, power_summary, ... }, "normal")
receive_directive()

Poll the operator endpoint for pending directives. Returns all queued directives sorted by priority. Called by the head on a configurable schedule and on reconnect events.

Returns: Directive[] — each with { type, priority, payload, issued_at_ms }
Example: receive_directive()
Latency: network round-trip
get_network_status()

Return current network connectivity state, queue depth, last successful uplink timestamp, and next scheduled transmission. Used by the head to decide whether to delay non-critical uplinks.

Returns: { connected: bool, queue_depth: int, last_uplink_ms: int, next_uplink_ms: int }
peer_announce()

Broadcast a presence announcement on the local network for peer submarine discovery. Returns a list of peers that responded, with their current observation status and RFI mask version.

Returns: Peer[] — each with { id, address, active_freq_mhz, rfi_mask_version }
Example: peer_announce()

Failure modes

What breaks.
How it recovers.

network down

Uplink endpoint unreachable

All summaries are queued locally. Arm 06 backs off exponentially (10s, 30s, 2min, 10min) to avoid hammering a flaky connection. The head is notified on the first failure and on reconnect. Science operations are not interrupted — the submarine operates fully autonomously offline.

queue overflow

Offline queue exceeds maximum depth

Arm 06 drops the oldest summaries first — recent data is more valuable than historical. It alerts the head with the queue depth and estimated data loss. The head may request an emergency uplink attempt or simply accept the loss depending on mission priorities.

auth failure

Operator endpoint rejects credentials

Arm 06 stops attempting uplink to avoid leaking data to an unexpected endpoint. It alerts the head and pauses all outbound communication until the credentials are updated via a local configuration command. Inbound directive polling also pauses.

malformed directive

Received directive fails schema validation

The directive is rejected and logged with the full payload for operator debugging. Arm 06 requests an acknowledgment from the server confirming the directive was received on its end, to prevent re-delivery loops. Other queued directives are processed normally.

External
