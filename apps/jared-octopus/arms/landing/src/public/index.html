What we built

A desk tool that reasons about the universe.

Submarine is an open-source multi-agent radiotelescope. Six OpenClaw instances operate as independent arms — each one a full AI agent with its own memory, toolset, and MCP server. A seventh acts as the head: decomposing observation goals into arm-level tasks, arbitrating conflicts, keeping the science honest.

The six arms

Each arm is a full agent. Not a function. Not a script.

Every arm runs an independent OpenClaw instance with its own MCP server, its own memory namespace, and a purpose-built toolset. The head sends high-level directives. Arms figure out how. If arm-01 wants to swap antennas while arm-02 is mid-observation, the head arbitrates — not the arms. Arms never talk to each other.

ARM 01

Antenna Positioning

Physical world interface

Controls servo motors and stepper drivers to physically rotate and position the antenna array. Executes swap sequences, validates mechanical confirmation via encoder feedback, and reports positioning errors upstream.

  • PCA9685 servo controller over I²C
  • Stepper driver with closed-loop encoder
  • MCP tool: swap_antenna(target, verify)
  • MCP tool: get_position()
  • Failure mode: parks antenna at safe azimuth
ARM 02

Signal Acquisition

Eyes on the sky

Drives the SDR hardware to capture raw IQ samples at a target frequency. Runs a real-time FFT pipeline to extract power spectral density. Knows when a signal is worth logging vs. discarding. Streams findings live to the data logger.

  • RTL-SDR v4 or HackRF One
  • GNU Radio FFT pipeline, 2.4 MSPS
  • MCP tool: observe(freq_mhz, bw, duration_sec)
  • MCP tool: scan_band(start, end, step)
  • Auto-notch for RFI rejection
ARM 03

Power Management

Keep the lights on

Monitors current draw on every rail. Maintains an energy budget across all active arms. When the budget is tight, it throttles non-critical arms rather than hard-cutting. Reports thermal state and flags anomalies before they become failures.

  • INA219 current + voltage sensors per rail
  • 200W bench supply coordination
  • MCP tool: power_status()
  • MCP tool: throttle(arm_id, level)
  • Thermal watchdog with safe-shutdown path
ARM 04

Data & Memory

Nothing is forgotten

Persists every observation: raw IQ samples, spectral summaries, timestamps, antenna position, power state, and agent reasoning traces. Provides a query interface that the head uses to build context before issuing new directives. Manages storage lifecycle.

  • SQLite for structured metadata
  • Flat files for raw IQ (GZIP compressed)
  • MCP tool: log(observation, metadata)
  • MCP tool: query(filter, limit)
  • Rolling 30-day retention with export
ARM 05

Calibration

Science needs ground truth

Runs periodic noise floor measurements, gain curve sweeps, and baseline drift corrections. Injects known reference signals to validate the signal path end-to-end. Publishes calibration deltas that arm-02 applies before each observation session.

  • Noise source injection via RF switch
  • Gain vs. temperature characterisation
  • MCP tool: calibrate(mode)
  • MCP tool: get_correction_table()
  • Runs at session start and on drift alert
ARM 06

Comms & Uplink

The submarine surfaces

Packages observation summaries and anomaly reports for uplink. Receives new mission directives from external operators or other submarine nodes. Acts as the network boundary — everything inbound and outbound flows through here. Supports peer-to-peer between submarine units.

  • HTTP/2 uplink to operator dashboard
  • MQTT for lightweight telemetry
  • MCP tool: uplink(summary, priority)
  • MCP tool: receive_directive()
  • Offline queue with retry on reconnect

Why MCP

The protocol choice determines the physics of the system.

CLI
Insufficient.

A new process per command. 150ms+ fork overhead. No streaming. When arm-02 is capturing 2.4 million samples per second, CLI is not a protocol — it's a bottleneck.

REST API
Partial.

Request and response. State disappears between calls. Polling wastes the same energy budget the power management arm is working to protect. Not built for continuous agents.

MCP
Persistent.

One socket per arm. Always open. State lives in the server. Sensor data streams up. Directives stream down. Arms and head stay in continuous contact — exactly like a nervous system should.

How it thinks

The head doesn't do science.
It decides who does.

The head is an OpenClaw operator. It holds the mission context, decomposes observation goals into arm-level tasks, and resolves conflicts. Arms never communicate directly — everything routes through the head. The graph stays acyclic. Science stays reproducible.

01

Acyclic graph only

Arms never message each other. Head receives, decides, dispatches. No circular dependencies. No emergent chaos.

02

Isolated memory namespaces

Each arm has its own memory. The head has its own. Cross-arm context only flows through the head's reasoning layer.

03

Modular by design

Swap arm-02 to a HackRF. Upgrade the head model. Change the uplink protocol in arm-06. Nothing else breaks.

04

Energy-aware scheduling

The head queries arm-03 before dispatching energy-intensive tasks. 150W budget. Respected by the architecture, not just the hardware.
