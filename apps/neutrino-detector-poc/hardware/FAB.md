# Fab Workflow

How to turn `NN-board/board.kicad_pcb` into files a manufacturer can build from.

## What manufacturers actually want

| File | What it is | Generated by |
|------|------------|--------------|
| **Gerbers** (`*.gbr`, one per layer) | The image of each copper / silk / mask / paste / outline layer | `kicad-cli pcb export gerbers` |
| **Drill file** (`*.drl`) | Hole positions and sizes (Excellon format) | `kicad-cli pcb export drill` |
| **Pick-and-place** (`*-cpl.csv`) | Component XY + rotation, for SMT assembly | `kicad-cli pcb export pos` |
| **BOM** (`BOM.csv`) | Designator, value, footprint, LCSC part number | already in each `NN-*/BOM.csv` |

For JLCPCB SMT assembly, you upload **Gerber zip + BOM.csv + CPL.csv**. They do the rest.

## Pipeline

After Adam saves a real schematic + PCB in KiCad, run:

```bash
cd apps/neutrino-detector-poc/hardware

# All five boards
./make-fab.sh

# Just one board
./make-fab.sh 01           # matches "01-sipm-frontend"
./make-fab.sh 01 02        # multiple
```

Each board ends up with:

```
NN-board-name/
├── fab/
│   ├── *.gbr                   ← one per copper/silk/mask/paste/outline layer
│   ├── *.drl                   ← drill file (Excellon, mm, abs origin)
│   ├── *-drl_map.pdf           ← drill map for visual sanity check
│   └── *-cpl.csv               ← pick-and-place
└── NN-board-name-fab.zip       ← upload-ready bundle
```

## Verify before sending

Three checks, in order from cheapest to most thorough:

1. **JLCPCB's web Gerber viewer.** Drag the zip in at the quote page; their renderer catches misaligned layers, missing outline, paste-on-silk errors. Free, instant.
2. **KiCad's gerbview** (`File → Open Gerber Plot File`). Catches the same issues offline, plus drill alignment against pads.
3. **Run DRC clean** in pcbnew before exporting:
   ```bash
   kicad-cli pcb drc --output drc-report.txt board.kicad_pcb
   ```
   Don't ship anything with DRC violations.

## JLCPCB (the recommended path for this PoC)

1. https://jlcpcb.com → "Quote Now" → upload `NN-board-name-fab.zip`
2. Their viewer renders the layers — eyeball it
3. Order options for our boards:

| Parameter | Value | Notes |
|-----------|-------|-------|
| Layers | 4 (or 2 for board 05) | matches design |
| Thickness | 1.6 mm | standard |
| Material | FR-4 | default |
| Surface finish | LeadFree HASL | cheapest viable; ENIG (~+$3) for board 01's SiPM pads if flatness matters |
| Solder mask | green | default; coloured is +$10 and slower |
| Silkscreen | white | default |
| Min order | 5 PCBs per design | ~$2 each at our sizes |
| Stencil (SMT) | yes if doing reflow yourself; no if JLCPCB assembles | |

4. **SMT assembly (optional):** upload `BOM.csv` (per-board, already in repo) + `*-cpl.csv` (generated by `make-fab.sh`). Choose top-side only for most; mark which components LCSC sources from "Basic" library (no extra setup fee) vs "Extended" (small fee per unique part).

## OSHPark (the cleaner alternative)

Higher quality, slower, more expensive. Dark purple solder mask by default. Their site accepts the same Gerber zip. Useful for the very first prototype of board 01 where SiPM-amplifier crosstalk matters more than cost.

## Aisler (EU-based, faster shipping to PL)

In-EU fab, ~5-day delivery to Poland vs ~10 from JLCPCB. ~2× the price. Drop the same zip at https://aisler.net.

## Design rules to stay inside (JLCPCB capability)

| Parameter | JLCPCB min | Our target (margin) |
|-----------|-----------|---------------------|
| Trace width | 0.127 mm | ≥ 0.15 mm |
| Trace spacing | 0.127 mm | ≥ 0.20 mm |
| Via diameter | 0.45 mm | ≥ 0.50 mm |
| Drill hole | 0.20 mm | ≥ 0.30 mm |
| Annular ring | 0.075 mm | ≥ 0.10 mm |
| Board outline | up to 400 × 500 mm | we're under 100 × 100 mm |

Stay above the "Our target" column on every board so fab tolerance doesn't bite.

## Skeleton vs real

Running `make-fab.sh` against the current v0.1 skeletons will produce:

- `Edge.Cuts.gbr` (board outline)
- `F.Silkscreen.gbr` (title text)
- `*.drl` (mounting holes only)
- No copper, no paste, no mask features

That proves the pipeline works but it's not buildable. Real Gerbers come after schematic capture + footprint placement + routing in KiCad.

## Provenance

The fab zip you upload should be reproducible from the `.kicad_pcb` in this repo at a known commit. Tag the commit and reference the SHA on the order email:

```bash
git tag pcb/01/v1.0
git push --tags
# Then in your JLCPCB order notes:
#   "Submarine repo @ commit a1b2c3d / tag pcb/01/v1.0"
```

That way, three months later you can always rebuild the exact same boards.

## Reproducibility — local Docker build

The fab pipeline runs locally, no third-party CI required. Two paths:

**Path A — host has KiCad installed** (fastest):
```bash
cd apps/neutrino-detector-poc/hardware
make fab            # all 5 boards
make fab-01         # just board 01
```

**Path B — host doesn't have KiCad** (Docker fallback, identical output):
```bash
cd apps/neutrino-detector-poc/hardware
make image          # one-off: ≈3 min, ≈500 MB image (Ubuntu 24.04 + KiCad 9.0)
make fab            # then identical to Path A
```

The Makefile auto-detects `kicad-cli` on `PATH` and picks the right runner. Run `make info` to see which path will be used.

Both paths produce byte-identical output (same KiCad version, same script, same `.kicad_pcb`). The Docker image is what makes this reproducible across your laptop, the Hetzner box, or anywhere else — no dependency on any third-party service being online.

### Why local instead of GitHub Actions

GitHub has frequent partial outages. Fab artefacts are produced infrequently (maybe 5–10 times a year, on the order of "we're about to spend money") and you don't want the build path blocked by a CI provider's status page. The Docker image lives on disk; `make fab` either succeeds or fails on your own hardware, independent of anyone else's infrastructure.

If you ever want push-to-build automation, the same Makefile + Dockerfile drops into any self-hosted runner (Forgejo Actions, Drone, a cron job on the Hetzner box) without changes.
